容器化是将应用程序及其所需的所有组件（如库、框架、配置文件）封装成一个独立的可执行单元——容器，使其在任何环境中都能一致运行。容器共享主机操作系统内核，但在独立的用户空间中运行，保证了进程隔离和环境一致性。这种方式比传统虚拟机更轻量，因为不需要为每个容器启动完整操作系统。
___
# 命名空间（Namespace）
在现代Linux中，命名空间是Linux内核提供的一种资源隔离机制。它将系统的全局资源划分为多个独立的空间，使得每个空间内的进程只可以看到和操作自己命名空间内的资源，实现“隔离”效果。

命名空间的核心目的：**让一个进程在"受限"的环境中运行，仿佛用于一个独立的系统资源。**  

命名空间是容器化技术的基石，docker中使用命名空间实现容器之间的资源隔离。
___
# 资源隔离（Cgroups）
Cgroups全称Control Groups，是Linux内核提供的物理资源隔离机制，通过这种机制，可以实现对Linux进程或者进程组的资源限制、隔离和统计功能。

比如可以通过Cgroups限制特定进程的资源使用，比如使用特定数目的CPU核数和特定大小的内存，如果资源超限的情况下，会被暂停或者杀掉。

Cgroups是于2.6内核由Google公司主导引入的，它是Linux内核实现资源虚拟化的技术基石，LXC(Linux Containers)和docker容器所用到的资源隔离技术，正是Cgroups。

# 镜像（Image）
镜像是一个只读的模板，包含运行应用所需的代码、运行时、库、环境变量和配置文件等内容，镜像本身是不可变的，容器是镜像的运行实例，拥有独立的进程空间、网络和文件系统，但共享宿主机的内核。

通俗来说，镜像就是容器的源代码，运行容器是基于镜像的。

镜像的核心特点
- 只读性。镜像是静态的，在运行容器时不会修改镜像，所以说是只读的（如果修改了镜像，就是在原有镜像的基础之上创建了一个新的镜像）。
- 分层存储。镜像采用分层存储的结构。
	- 在构建时，不必重新构建未改动的层级，复用之前的即可。
	- 传输时，也可以避免重复层级的传输。
	- 降低空间消耗，。如果两个容器的底层是基于相同的操作系统，那么就可以复用这个os层，避免重复层间占用空间。
- 可移植。镜像是一个单元，可以被打包，传输、管理，在任何支持docker的环境中运行。
___
# 容器
容器是镜像的运行实例，是独立运行的一个或一组应用，以及它们的运行环境。

容器和镜像的区别在于，镜像是静态，容器是运行态，在镜像的基础层级之上添加了容器层（可读写层），每一个容器都有单独的容器层，容器的运行数据都存储在该层级中。该层级和容器的生命周期保持一致，一起创建，一起销毁，也就意味着容器被销毁了，数据也会随之丢失，这种情况会使用数据卷（Volume）去进行解决。
___
# 数据卷（Volume）
*“Volume 是供容器使用的持久化数据存储，由docker创建并管理。”* 

这是docker官方对于数据卷的定义，简单来说，数据卷就是独立于容器层的存储空间。其目的是为了将数据与服务分离开，容器的理念是无状态的，意味着可以随便创建，随便销毁，因此需要将数据与服务分离开。
