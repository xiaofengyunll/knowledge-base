# 模块一：软件测试基本概念

## 核心知识点

### 1. 软件测试的目的与原则

#### 测试目的
- 发现缺陷（Bug）
- 验证软件是否满足需求
- 提高软件质量

#### 测试原则
- 尽早测试
- 经济性原则
- 全面性原则
- 系统性原则
- 客观性原则

### 2. 软件的组成
- 需求
- 设计
- 编码
- 测试
- 维护

### 3. 验证与确认

#### 验证（Verification）
确保产品“按照需求正确实现”。

#### 确认（Validation）
确保产品“真正满足用户需求”。

### 4. V模型

V模型是一种软件开发模型，强调：
- 开发与测试同步进行
- 每个开发阶段对应一个测试阶段

常见对应关系：

| 开发阶段 | 对应测试阶段 |
|---|---|
| 需求分析 | 验收测试 |
| 系统设计 | 系统测试 |
| 详细设计 | 集成测试 |
| 编码 | 单元测试 |

## 复习要点
- 理解软件测试的目的和原则
- 掌握验证与确认的区别
- 理解V模型的意义

# 模块二：缺陷管理

## 核心知识点

### 1. 缺陷（Bug）的定义
软件产品或系统中的错误、缺陷或问题。

### 2. 缺陷生命周期

常见状态：
1. 未报告
2. 已报告
3. 已修复
4. 已验证
5. 已关闭

### 3. 缺陷修复成本

特点：
- 缺陷发现越早，修复成本越低
- 后期修复成本会显著增加

## 复习要点
- 理解缺陷生命周期
- 掌握缺陷修复成本的重要性

# 模块五：测试驱动开发（TDD）

## 核心知识点

### 1. TDD基本流程（红-绿-重构）

#### 红（Red）
先编写失败的测试代码。

#### 绿（Green）
编写最少功能代码使测试通过。

#### 重构（Refactor）
优化代码结构并保证测试继续通过。

### 2. TDD的特点
- 先写测试代码，再写功能代码
- 主要用于单元测试阶段
- 所有测试通过后才能重构

## 复习要点
- 理解TDD核心思想
- 掌握TDD与传统开发方式的区别

# 模块六：黑盒测试设计技术（理论部分）

## 核心知识点

### 1. 等价类划分

#### 有效等价类
符合规格说明的输入。

#### 无效等价类
违反规格说明的输入。

### 2. 边界值分析
- 边界值最容易出错
- 重点测试：
  - 最大值
  - 最小值
  - 刚超过边界
  - 刚低于边界

### 3. 决策表法
适用于：
- 多条件组合逻辑场景

组成：
- 条件项
- 动作项
- 决策规则

### 4. 因果图
用于处理：
- 多条件之间的逻辑关系

## 复习要点
- 掌握等价类划分方法
- 理解边界值分析的重要性
- 理解决策表法和因果图的应用

# 模块七：白盒测试逻辑覆盖（理论部分）

## 核心知识点

### 1. 语句覆盖
确保每条语句至少执行一次。

### 2. 判定覆盖
确保每个判定结果：
- True
- False

都至少执行一次。

### 3. 条件覆盖
确保每个条件都至少取一次：
- 真
- 假

### 4. 路径覆盖
确保所有可能执行路径都被测试。

## 复习要点
- 掌握不同逻辑覆盖标准
- 理解白盒测试覆盖思想

# 模块八：性能测试

## 核心知识点

### 1. 性能测试的目的
- 评估系统性能
- 识别性能瓶颈

### 2. 性能测试类型

#### 负载测试
验证正常负载下系统性能。

#### 压力测试
验证超负载情况下系统表现。

#### 稳定性测试
长时间运行测试系统稳定性。

#### 并发测试
测试多用户同时访问能力。

### 3. 性能测试指标
- 响应时间
- 吞吐量
- 并发用户数
- 资源利用率（CPU、内存等）

## 复习要点
- 理解性能测试目的与类型
- 掌握关键性能指标
