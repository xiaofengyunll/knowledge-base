# 基本概念
## 特点
- 覆盖较小的地理范围
- 较低的时延和误码率局域
- 网内的各节点之间“帧”为单位进行传输
- 支持单播（一对一发送帧）、广播（一对全部发送帧）、多播（一对部分发送帧）

## 分类
#### 有线局域网（LAN）

**以太网**（物理层采用曼彻斯特编码）
- 同轴电缆以太网（早期，1980年开始商用）
	- 总线形
	- 同轴电缆（可用中继器连接多个同轴电缆网段）
	- CSMA/CD协议
- 双绞线以太网
	- 用集线器连接（半双工）
		- 物理上星形，逻辑上总线形
		- 双绞线（用集线器连接）
		- CSMA/CD协议
	- 用交换机连接（1994年后逐步转向交换机）
		- 物理上、逻辑上都是星形
		- 双绞线（用交换机连接）
		- CSMA/CD协议（半双工）或不考虑介质访问控制（全双工）
- 光纤以太网
	- 点对点（用于中继器/集线器/交换机之间的传输，也就是说通常不会直接连接终端节点）
	- 光纤
	- 不考虑介质访问控制（用两条光纤实现全双工通信）

注：CSMA/CD 是数据链路层 MAC 子层协议；集线器不执行该协议，但集线器构成共享介质网络，碰撞检测和退避由连接到集线器的网卡完成。

**令牌环网**
- 环形
- 同轴电缆或双绞线
- 令牌传递协议

#### 无线局域网（WLAN）
**WiFi/802.11**
- 802.11定义为星形（1个AP+N台移动没备）
- 无线（Wireless）
- CSMA/CA 协议

# 硬件架构
## 网络适配器（网络接口卡、网卡）
- 负责把帧发送到局域网。注:关于“将IP数据报(IP分组)封装成帧”，有些系统由主机实现，有些系统由网络适配器负责。
- 负责从局域网接收帧。如果收到正确帧，就用“中断"通知CPU;如果收到异常帧，就直接丢弃
- 需要根据接入的局域网类型，按照标准实现数据链路层+物理层功能
- 需要完成数据的串/并行转换，曼彻斯特串行通讯，I/O总线并行通讯
- 需要支持帧缓冲

**MAC 地址**
- 物理地址，固化在网络适配器的 ROM（只读寄存器）中
- 全球唯一
- 48bit（高24bit为厂商号，低24bit由厂商分配）

注：MAC地址是数据链路层的概念。路由器、交换机都有MAC地址，而集线器没有MAC地址。

RAM：用于帧缓冲
