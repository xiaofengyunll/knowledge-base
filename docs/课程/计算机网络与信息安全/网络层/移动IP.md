# 基本概念
**移动节点/移动站**：具有永久IP地址的移动主机（就是你的手机，拥有和电话卡绑定的专属IP地址）。

**归属代理/本地代理**：通常就是连接在归属网经（原始连接到的网络）上的路由器，负责代收数据并通过“隧道”转发给移动站的新位置。

**外地代理**：通常就是连接在被访网络（移动到另一地点所接入的网络）上的路由器。

**漫游**：一个插着北京电话跑到外地去了，通过“外地代理”实现上网。

**永久地址**：和手机电话号码绑定的，全球唯一的、永久的IP地址。

**转交地址**：手机漫游到外地网络时，被分配的临时IP地址。

---
# 移动IP通信过程
**移动站在外地接收IP分组**
- 移动站A漫游到被访网络时，向外地代理进行登记，以获得一个临时的转交地址。外地代理要向A的归属代理登记A的转交地址。
- 归属代理知道移动站A的转交地址后，会构建一条通向转交地址的隧道，将截获的发送给A的IP分组进行再封装，并通过隧道发送给被访网络的外地代理。

**移动站在外地发送IP分组**
移动站A在被访网络对外发送IP分组时，仍然使用自己的永久地址作为IP分组的源地址，此时显然无须通过A的归属代理来转发，而是直接通过被访网络的外部代理。

**移动站回到归属网络**
移动站A回到归属网络时，A向归属代理登记，注销转交地址。

注：
1. 转交地址是供移动站、归属代理及外地代理使用的。移动站与其他主机之间的通信，都只使用“永久地址。
2. 一个转交地址可能被多个移动站共用。外地代理要向连接在被访网络上的移动站发送IP分组时，直接使用移动站的MAC地址。

# 网络层支持
- 移动站到外地代理的登记协议；
- 外地代理到归属代理的登记协议；
- 归属代理数据报封装协议；
- 外地代理拆封协议。
