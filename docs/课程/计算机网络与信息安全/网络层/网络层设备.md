**路由器的控制层面**
- **路由处理机**：相当于路由器的CPU，运行路由选择协议进程（如：RIP/OSPF/BGP进程），生成路由表（Routing Table）。
- **路由表表项**：少包含【网络前缀，下一跳IP，出接口】。
- 如果路由器的下一跳就是目的网络，称为“直接交付“，下一跳IP在底层二进0制通常写为0.0.0.0。
- 如果IP分组的下一跳是路由器，称为“间接交付”。

**路由器的数据层面**
- 每个输入端口网络层处理模块设有“输入缓冲区”，用于暂存交换结构来不及处理的IP分组分组。如果输入缓冲区满，但仍有新的IP分组到达，则新分组丢失（发生网络拥塞）。
- 每个输出端口网络层处理模块设有“输出缓冲区”，用于暂存交换结构已处理（已确定输出端口），但是由于带宽限制暂时无法发送到链路上的IP分组。
