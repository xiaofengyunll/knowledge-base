# 数据库讲解大纲（2 小时概要版 · 第一节课）

## 一、数据库基础概念（约 15 分钟）

### 1.1 基本认识

- 数据、数据库、数据库管理系统（DBMS）、数据库系统的区别
- 文件系统 vs 数据库系统的对比
- 数据库系统的特点：数据共享、减少冗余、统一管理、安全控制

### 1.2 数据库系统结构

- 数据库系统组成：用户 → 应用程序 → DBMS → 数据库
- 三级模式结构（简要）：
    - 外模式（用户视图）
    - 模式（逻辑结构）
    - 内模式（物理存储）
- 两级映像与数据独立性（逻辑独立性、物理独立性）

---

## 二、关系模型基础（约 15 分钟）

### 2.1 数据模型概述

- 概念模型 → 逻辑模型 → 物理模型
- 常见模型：层次模型、网状模型、**关系模型**（当前主流）

### 2.2 关系模型核心概念

- 关系 = 二维表
- 元组（行）、属性（列）、域、候选码、主码、外码
- 关系模式 vs 关系实例

### 2.3 完整性约束

- 实体完整性（主键非空且唯一）
- 参照完整性（外键约束）
- 用户自定义完整性

---

## 三、SQL 基础与应用（约 45 分钟）

> 本节以实操为主，建议讲师边讲边演示。

### 3.1 SQL 概述

- SQL 分类：
    - DDL：数据定义（CREATE、DROP、ALTER）
    - DML：数据操作（INSERT、UPDATE、DELETE）
    - DQL：数据查询（SELECT）
    - DCL：数据控制（GRANT、REVOKE）

### 3.2 数据库与表操作

- 创建/删除数据库与表
- 常见数据类型：整数、小数、字符串、日期时间
- 常见约束：PRIMARY KEY、FOREIGN KEY、NOT NULL、UNIQUE、DEFAULT

### 3.3 数据增删改（CRUD）

- INSERT 插入数据
- UPDATE 修改数据
- DELETE 删除数据
- DELETE 与 TRUNCATE 的区别

### 3.4 单表查询

- SELECT + FROM + WHERE 基本语法
- 条件筛选：BETWEEN、IN、LIKE、IS NULL
- ORDER BY 排序、LIMIT 分页、DISTINCT 去重

### 3.5 聚合查询

- 聚合函数：COUNT、SUM、AVG、MAX、MIN
- GROUP BY 分组
- HAVING 分组后筛选（与 WHERE 的区别）

### 3.6 多表查询

- 内连接 INNER JOIN
- 左/右外连接 LEFT JOIN / RIGHT JOIN
- 子查询
- EXISTS 与 IN 的区别

## 四、E-R 模型与表设计入门（约 15 分钟）

### 4.1 数据库设计流程概述

- 需求分析 → 概念设计 → 逻辑设计 → 物理设计 → 实施维护

### 4.2 E-R 模型

- 实体、属性、联系
- 一对一、一对多、多对多关系
- E-R 图向关系模式的转换思路

### 4.3 表设计基本准则

- 字段命名规范、主键设计原则
- 字段类型选择原则
- 外键与关联设计

---

## 五、索引基础（约 15 分钟）

### 5.1 什么是索引

- 索引的作用：加速查询（类比书的目录）
- 索引的代价：占用空间、降低写入速度

### 5.2 常见索引类型

- 主键索引、唯一索引、普通索引、联合索引
- 聚簇索引 vs 非聚簇索引
- 回表查询与覆盖索引

### 5.3 索引使用原则

- 最左前缀原则
- 适合建索引的场景
- EXPLAIN 分析执行计划（简要演示）

---

## 六、事务基础（约 10 分钟）

### 6.1 事务概念

- 事务的定义：一组不可分割的操作
- 基本操作：BEGIN、COMMIT、ROLLBACK

### 6.2 ACID 特性

- 原子性、一致性、隔离性、持久性

### 6.3 并发问题与隔离级别

- 脏读、不可重复读、幻读、丢失更新
- 四种隔离级别（简要介绍）
- 隔离级别越高，一致性越好，并发性能越低

---

### 7.1 本节回顾

- 数据库是什么 → 关系模型 → SQL 怎么用 → 表怎么设计 → 索引做什么 → 事务保什么
