# 什么是DevOps文化?
DevOps文化的本质是实现组织变革的一种敏捷开发方法，旨在打破传统开发与运维之间的隔阂，建立并促进协作的新流程。

传统软件开发中开发追求快速交付与运维追求稳定的冲突中，易出现开发与运维相会甩锅的情况，导致责任划分出现问题，产品质量也受到影响。

DevOps要求进行共担责任，开发人员需要贯彻“谁构建，谁运行”的理念，不仅需要完成构建移交给运维团队，还要肩负起整个产品生命周期内的产品监督责任。开发团队在开发和测试软件基础上，还需要与测试团队、运维团队保持密切的沟通协作。

DevOps注重快速反馈，有助于产品的持续改进，保证交付质量。传统开发链式中，开发团队和运维团队孤立的开展工作，导致生产环境中性能和稳定性反馈通常较慢。DevOps 运营人员在设计和实施应用监控和报告策略时进行协作，确保开发人员能够快速得到反馈，进行快速迭代和应用代码改进。

自动化对于DevOps来说至关重要，可以通过自动化来实现不同团队之前的高效协作
