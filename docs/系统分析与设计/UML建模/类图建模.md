类图用于描述系统的静态结构，包括：
- 系统中有哪些类或接口；
- 类具有哪些操作；
- 类之间的关系；
- 每个关联端允许连接多少个对象。

**可见性**
- +：public，共有；
- -：private，私有；
- \#，protected，受保护；
- ~，package，包内可见。

**类之间的关系**
1. 依赖
	- 含义：一个类中临时使用另一个类
	- UML：虚线箭头指向被依赖者。
2. 关联
	- 含义：对象之间存在相对稳定的结构性连接
	- UML：实线，可标角色名、导航方向（箭头）和多重度。
3. 聚合
	- 含义：较弱的整体/部分关系，部分可独立于整体存在。
	- UML：空心菱形位于整体端。
4. 组合
	- 含义：较强的整体/部分关系，部分生命周期依赖整体。
	- UML：实心菱形位于整体端。
5. 泛化（继承）
	- 含义：一般类与特殊类的“is_a”关系，即继承关系。
	- UML：实线空心三角箭头指向父类。
6. 实现
	- 含义：类实现接口声明的服务。
	- UML：虚线空心三角箭头指向接口。
