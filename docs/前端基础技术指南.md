# 前端基础技术指南

HTML5 语义化、CSS3 核心机制、以及从传统浮动到现代 Flex/Grid 的页面布局方案。

---

## 一、HTML5：语义化与文档结构

### 1.1 文档骨架

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>文档标题</title>
</head>
<body>
    <!-- 页面内容 -->
</body>
</html>
```

`viewport` meta 标签是响应式的起点。没有它，移动端浏览器会以桌面宽度（约 980px）渲染页面然后缩小，文字小到需要放大才能阅读。

### 1.2 语义化标签

| 标签 | 用途 |
|---|---|
| `<header>` / `<footer>` | 页头/页脚，不限于页面级别，`<article>` 内部也可以有自己的 header/footer |
| `<nav>` | 主导航链接集合 |
| `<main>` | 页面主体内容，一个文档只出现一次 |
| `<article>` | 独立、可分发的内容块（博客正文、评论卡片） |
| `<section>` | 按主题分组的区域，通常带标题 |
| `<aside>` | 侧边栏或补充内容（广告、相关链接） |

语义化的直接收益：屏幕阅读器能跳过导航直达 `<main>`，搜索引擎更准确理解页面骨架。比 `<div class="header">` 多了机器可读的结构信息。

### 1.3 表单增强

HTML5 新增的 input 类型和属性减少了对 JavaScript 验证的依赖：

```html
<!-- 类型约束 -->
<input type="email" required placeholder="you@example.com">
<input type="number" min="1" max="100" step="5">
<input type="date">
<input type="range" min="0" max="100">

<!-- 手机端唤起对应键盘 -->
<input type="tel" pattern="1[3-9]\d{9}">
<input type="url">

<!-- 原生表单验证 -->
<form novalidate>
    <input type="text" required minlength="3" maxlength="20">
</form>
```

实际开发中注意事项：

- `pattern` 属性在前端只是辅助校验，后端必须再次验证。
- `required` 配合 `:invalid` / `:valid` 伪类可以做即时反馈，但如果样式太激进反而干扰用户——建议在 blur 或 submit 时才展示校验状态。
- `novalidate` 关闭浏览器默认校验气泡，适合用自定义 UI 替代。

### 1.4 多媒体

```html
<!-- 音频：controls 开启原生播放控件 -->
<audio controls preload="metadata">
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    你的浏览器不支持音频播放。
</audio>

<!-- 视频 -->
<video controls width="100%" poster="cover.jpg">
    <source src="video.mp4" type="video/mp4">
</video>
```

`preload="metadata"` 只预加载元信息（时长、尺寸），在移动端可以节省流量。`poster` 属性设置视频加载前的封面图，避免页面出现一个黑框。

## 二、CSS3 核心机制

### 2.1 选择器优先级（特异性）

浏览器通过一套计分规则决定冲突样式谁生效：

```
!important > 内联 style > ID > 类/属性/伪类 > 元素/伪元素
```

实际开发中最常见的优先级问题不是忘记规则，而是在组件嵌套变深后，无意中写高了选择器特异性，后续覆盖只能继续加层级，形成"选择器军备竞赛"。

**保持低特异性**的建议：用单一 class 选择器（BEM 命名），只在需要覆盖第三方样式时才动用多层选择器。

### 2.2 盒模型

```css
/* 标准盒模型：width = content */
.box { box-sizing: content-box; }

/* 替代盒模型：width = content + padding + border */
.box { box-sizing: border-box; }
```

几乎所有项目的全局样式中都会有：

```css
*, *::before, *::after {
    box-sizing: border-box;
}
```

这样做是因为 `border-box` 更符合直觉：你设置 `width: 300px`，元素占的宽度就是 300px，不会因为 padding 和 border 超出预期。

### 2.3 层叠上下文（Stacking Context）

`z-index` 不是全局生效。它只在同一个层叠上下文内比较。下面这些属性会创建新的层叠上下文：

- `position: relative/absolute/fixed/sticky` + `z-index` 不为 `auto`
- `opacity` 小于 1
- `transform`、`filter`、`perspective` 不为 `none`
- `flex` / `grid` 容器的子项且 `z-index` 不为 `auto`

开发中最常见的坑：给父元素加了一个细微的 `opacity: 0.99`（想表达"半透明"但没有真正透明），结果内部弹窗的 `z-index: 9999` 完全失效。

### 2.4 CSS 变量（自定义属性）

```css
:root {
    --color-primary: #3b82f6;
    --color-bg: #f9fafb;
    --spacing-unit: 8px;
    --radius-md: 6px;
}

.card {
    background: var(--color-bg);
    padding: calc(var(--spacing-unit) * 3);
    border-radius: var(--radius-md);
}

/* 暗色主题切换 */
[data-theme="dark"] {
    --color-bg: #1a1a2e;
    --color-primary: #818cf8;
}
```

相比 Less/Sass 变量，CSS 变量的优势在于它是**运行时动态的**——可以通过 JS 修改或在特定选择器下重定义，不需要重新编译。主题切换就是最典型的场景。

### 2.5 过渡与动画

```css
/* transition：状态变化时的平滑过渡 */
.button {
    background: #3b82f6;
    transition: background 0.2s ease, transform 0.15s ease;
}
.button:hover {
    background: #2563eb;
    transform: scale(1.02);
}

/* animation：独立的关键帧动画 */
@keyframes fadeIn {
    from { opacity: 0; transform: translateY(8px); }
    to   { opacity: 1; transform: translateY(0); }
}
.card--enter {
    animation: fadeIn 0.3s ease both;
}
```

选择建议：

- `transition` 适用于**用户交互触发的简单过渡**（hover、focus、class 切换）。
- `animation` 适用于**进入/离开动画**、循环动画（loading spinner）、复杂多阶段动画。
- 在移动端，优先对 `opacity` 和 `transform` 做动画——它们只触发合成（composite），不触发重排或重绘，性能最好。对 `width`、`height`、`top`、`left` 做动画会导致重排，触发整个渲染管线。

## 三、页面布局方案

### 3.1 浮动布局（Float）

浮动的原本目的是实现"文字环绕图片"的排版效果。在 Flex/Grid 出现之前，它是多栏布局的唯一手段——属于历史环境下的一种"借用"。

#### 3.1.1 浮动行为

元素设置 `float: left` 或 `float: right` 后：

1. 脱离正常文档流（但不完全脱离，文字会感知它）。
2. 向指定方向移动，直到碰到容器边缘或另一个浮动元素。
3. 后续的**行内元素和文本会环绕**浮动元素，但块级元素会穿过它。

#### 3.1.2 清除浮动

浮动元素不撑开父容器高度——因为浮动元素脱离了文档流，父元素在计算高度时看不到它。

**方法一：clearfix（最传统）**

```css
.clearfix::after {
    content: "";
    display: block;   /* 也可以是 table */
    clear: both;
}
```

原理：`::after` 伪元素创建了一个在浮动元素后面的块级元素，`clear: both` 让它出现在所有浮动元素下方，从而撑开父容器。

**方法二：overflow（简单但有副作用）**

```css
.container {
    overflow: hidden; /* 或 auto */
}
```

原理：`overflow: hidden` 触发 BFC（块级格式化上下文），BFC 容器在计算高度时会包含其浮动子元素。缺点——内容溢出会被裁剪，不能用于下拉菜单或 tooltip 场景。

**方法三：Flex/Grid 替代（现代方案）**

如果不需要兼容 IE，直接用 Flex 或 Grid 替代浮动布局。浮动从出生就不是为了布局设计的，用了 20 年代价的 hack。

#### 3.1.3 浮动在今天还有用吗？

有，但场景很窄：文字环绕图片、首字下沉效果。做多栏布局不应该再用浮动。

### 3.2 定位布局（Position）

#### 3.2.1 五种定位模式

| 值 | 参照物 | 是否脱离文档流 | 典型用途 |
|---|---|---|---|
| `static` | 默认值，正常流 | 否 | — |
| `relative` | 元素自身原本的位置 | 否（保留占位） | 微调位置，或作为 absolute 的锚点 |
| `absolute` | 最近的非 static 祖先 | 是 | 弹出层、下拉菜单、图标角标 |
| `fixed` | 视口（viewport） | 是 | 固定导航栏、返回顶部按钮、模态框遮罩 |
| `sticky` | 最近的滚动祖先 | 否（保留占位） | 吸顶导航、表格固定表头 |

#### 3.2.2 absolute 的锚点规则

```css
.parent { position: relative; } /* 不设置 top/left，元素位置不变，仅作为锚点 */

.child {
    position: absolute;
    top: 100%;
    left: 0;
}
```

关键点：`absolute` 参照"最近的非 static 祖先"。如果没有找到这样的祖先，则参照 `<html>`（即视口）。所以写下拉菜单时，在父元素（`<li>` 或 `<div>`）上设置 `position: relative` 作为锚定容器是标配操作。

#### 3.2.3 sticky 的生效条件

`position: sticky` 相比 fixed 更"聪明"，但有几个常被忽略的限制：

1. 必须至少指定一个偏移方向（`top`、`right`、`bottom`、`left`），否则和 `relative` 没区别。
2. 父元素的 `overflow` 不能是 `hidden` 或 `scroll`，否则 sticky 失效（实际上 sticky 受限于最近的滚动容器）。
3. 父元素高度不够时，sticky 元素可能看起来"没生效"——它不会超出父元素边界。

```css
.header {
    position: sticky;
    top: 0;
    z-index: 10;
    background: white; /* 必须设置，否则滚动时内容穿透 */
}
```

#### 3.2.4 定位 + 居中的经典组合

```css
/* 绝对定位 + transform 居中（内容不定位宽高时最常用） */
.modal {
    position: absolute; /* 或 fixed */
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}
```

原理：`50%` 是参照父容器的宽高，`translate(-50%, -50%)` 是参照元素自身的宽高。两者结合达到精确居中，不需要知道元素具体尺寸。

### 3.3 Flex 弹性布局

Flex 是 CSS 第一个真正为"布局"而设计的模块。它处理一维方向的排列和空间分配非常自然。

#### 3.3.1 容器属性

```css
.container {
    display: flex;               /* 或 inline-flex */
    flex-direction: row;          /* row | row-reverse | column | column-reverse */
    flex-wrap: nowrap;           /* nowrap | wrap | wrap-reverse */
    justify-content: flex-start; /* 主轴对齐 */
    align-items: stretch;        /* 交叉轴对齐（单行） */
    align-content: stretch;      /* 交叉轴对齐（多行） */
    gap: 16px;                   /* 子元素间距（推荐替代 margin） */
}
```

主轴和交叉轴的方向由 `flex-direction` 决定。`flex-direction: row` 时，主轴是横轴，交叉轴是纵轴；`flex-direction: column` 时互换。很多对齐表现"不合预期"都是因为搞混了主轴和交叉轴方向。

**`justify-content` 常见值：**

```
flex-start ─── 元素靠左
flex-end  ─────── 元素靠右
center   ─────── 居中
space-between ─ 两端对齐，间距均分
space-around ── 每个元素两侧间距相等（首尾间距是中间的一半）
space-evenly ── 所有间距相等（包括首尾）
```

`space-between` 适用于两端对齐的导航栏，`space-evenly` 和 `space-around` 适用于卡片列表。

**`gap` 替代 `margin` 的意义：**

```css
/* 旧方案：用 margin 做间距，需要处理最后一个元素 */
.item { margin-right: 16px; }
.item:last-child { margin-right: 0; }

/* Flex 方案：gap 只在元素之间生效，首尾没有多余间距 */
.container {
    display: flex;
    gap: 16px;
}
```

#### 3.3.2 子元素属性

```css
.item {
    flex-grow: 0;    /* 有多余空间时，放大比例 */
    flex-shrink: 1;  /* 空间不足时，缩小比例 */
    flex-basis: auto; /* 初始主轴尺寸 */
    flex: 1;          /* 缩写：flex: 1 1 0% */

    align-self: auto; /* 覆盖容器的 align-items */
    order: 0;         /* 排列顺序，越小越靠前 */
}
```

**`flex` 缩写的三种常用形态：**

| 写法 | 等价 | 行为 |
|---|---|---|
| `flex: 1` | `flex: 1 1 0%` | 均分剩余空间，自动缩放 |
| `flex: auto` | `flex: 1 1 auto` | 基于内容大小分剩余空间 |
| `flex: none` | `flex: 0 0 auto` | 不伸缩，保持内容尺寸 |
| `flex: initial` | `flex: 0 1 auto` | 不放大但允许缩小（默认值） |

`flex: 1` 和 `flex: auto` 的区别是一个常见的混淆点：前者从 0 开始均分空间（`flex-basis: 0%`），后者基于内容自身体积后再分剩余空间。大多数"等宽布局"用 `flex: 1`，内容自适应的侧边栏+主体布局用 `flex: auto`。

#### 3.3.3 常见布局模式

**等宽多栏：**

```css
.grid { display: flex; gap: 20px; }
.grid > * { flex: 1; }
/* 所有子元素等宽 */
```

**圣杯布局（三栏，中间自适应，两边固定）：**

```css
.layout { display: flex; }
.sidebar-left  { width: 240px; flex-shrink: 0; }
.main-content  { flex: 1; min-width: 0; }        /* min-width: 0 防止内容撑破 */
.sidebar-right { width: 300px; flex-shrink: 0; }
```

`min-width: 0` 是关键。Flex 子元素的默认 `min-width` 是 `auto`，意味着元素不会缩小到内容尺寸以下。如果 main-content 里放了一个很长的 URL（没有断行点），它会撑开整个容器。加上 `min-width: 0` 可以让它正常截断或换行。

**垂直居中：**

```css
.center {
    display: flex;
    justify-content: center;
    align-items: center;
}
/* 一行解决，比表格布局和绝对定位方案都简洁 */
```

**导航栏（logo 在左，菜单在中，操作在右）：**

```css
header {
    display: flex;
    align-items: center;
    padding: 0 24px;
}
header .logo    { margin-right: auto; }
header .actions { margin-left: auto;  }
/* margin auto 在 Flex 容器中会吃掉所有可用空间，把相邻元素推向两端 */
```

`margin: auto` 在 Flex 容器中的这个行为尤其实用：它按主轴方向吸收多余空间。比用 `justify-content: space-between` 包裹一层更灵活。

### 3.4 Grid 网格布局

Flex 处理一维布局（行或列），Grid 处理二维布局（行和列同时控制）。页面骨架级的分区方案、仪表板卡片排列、表单标签-输入框对齐——这些场景 Grid 比 Flex 更合适。

#### 3.4.1 定义网格

```css
.grid {
    display: grid;
    /* 三列：两边自适应，中间固定 */
    grid-template-columns: 1fr 640px 1fr;
    /* 间距 */
    gap: 24px 16px;  /* 行间距 列间距 */
    /* 隐式行高度 */
    grid-auto-rows: minmax(120px, auto);
}
```

`fr` 单位是 Grid 独有的：它代表"可用空间的一份"。`1fr 2fr 1fr` 等于 1:2:1 的列宽比。`fr` 和 `flex-grow` 思想类似，但用在网格轨道上。

**`minmax()` 的意义：** `grid-auto-rows: minmax(120px, auto)` 让每一行至少 120px，内容多时自动撑高。这替代了传统的 `min-height` + `height: auto` 的写法。

#### 3.4.2 网格线定位

每一条网格线都有编号（从 1 开始），也可以用 `grid-template-areas` 给区域命名：

```css
.layout {
    display: grid;
    grid-template-columns: 240px 1fr 300px;
    grid-template-rows: 60px 1fr 80px;
    grid-template-areas:
        "header  header  header"
        "sidebar main    aside"
        "footer  footer  footer";
    min-height: 100vh;
}

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main   { grid-area: main; }
.aside  { grid-area: aside; }
.footer { grid-area: footer; }
```

`grid-template-areas` 的可读性远高于用行号定位。修改布局骨的时候，调整区域名称的排列即可，不需要重新计算每个元素的 `grid-column` 和 `grid-row`。

#### 3.4.3 对齐

Grid 的对齐体系和 Flex 类似但多了一个维度：

```css
.grid {
    /* 网格整体在容器内的对齐 */
    justify-content: center;  /* 水平 */
    align-content: center;    /* 垂直 */

    /* 子元素在各自网格区域内的对齐 */
    justify-items: stretch;   /* 水平 */
    align-items: stretch;     /* 垂直 */
}

.item {
    /* 单个子元素覆盖 */
    justify-self: center;
    align-self: center;
}
```

#### 3.4.4 Grid vs Flex 的选择

| 场景 | 推荐 |
|---|---|
| 页面骨架分区（header/sidebar/main/footer） | Grid |
| 一维排列的元素列表（导航栏、标签页、卡片行） | Flex |
| 固定行列的仪表板、相册、产品网格 | Grid |
| 内容驱动的自动换行（标签云、筛选条件） | Flex + `flex-wrap: wrap` |
| 表单标签-输入框对齐（行列严格对应） | Grid |
| 垂直居中的弹窗内容 | Flex |

Flex 和 Grid 不互斥。页面骨架用 Grid，骨架内的组件用 Flex，是常见的组合方式。

### 3.5 响应式布局

#### 3.5.1 视口（Viewport）

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

这个标签不加的话，移动端 Safari/Chrome 会默认把页面当成 980px 宽的桌面页面渲染后再缩放——整个页面缩小到手机屏幕宽度，所有文字都不可读。`width=device-width` 告诉浏览器以设备的实际像素宽度渲染。

#### 3.5.2 媒体查询（Media Queries）

```css
/* 移动优先：基础样式是移动端的，用 min-width 逐级增强 */
.card { padding: 12px; }                    /* 手机 */

@media (min-width: 768px) {
    .card { padding: 20px; }                /* 平板 */
    .grid { grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {
    .card { padding: 24px; }                /* 桌面 */
    .grid { grid-template-columns: repeat(3, 1fr); }
}
```

**为什么用 min-width（移动优先）而不是 max-width（桌面优先）？**

移动优先的 CSS 更简洁：基础样式在移动端生效，媒体查询只负责在大屏幕上调整。反过来用 max-width 时，需要为移动端写大量覆盖样式。而且移动端设备种类远多于桌面端，让"最受限制的环境"拥有最简单的样式逻辑符合实际工程直觉。

**常见断点参考：**

| 断点 | 对应设备 |
|---|---|
| < 576px | 手机竖屏 |
| >= 576px | 手机横屏 |
| >= 768px | 平板 |
| >= 992px | 小屏笔记本 |
| >= 1200px | 桌面 |
| >= 1400px | 大屏桌面 |

这些数字不必严格遵循——断点应该由**内容**决定，而不是设备列表。当内容看起来"太挤"或"太稀"时加断点，比套用固定的设备宽度更合理。

#### 3.5.3 响应式工具函数（clamp / min / max）

```css
/* 标题随视口缩放，但限制在 18px ~ 48px 之间 */
.title {
    font-size: clamp(18px, 4vw, 48px);
}

/* 图片宽度不超过父容器，最小 200px */
.image {
    width: min(100%, 600px);
}

/* 容器宽度占父元素 80%，但不能少于 320px */
.container {
    width: max(80%, 320px);
}
```

`clamp()` 是响应式排版最实用的函数。传统做法是多层媒体查询在不同断点覆盖 `font-size`，`clamp(最小值, 首选值, 最大值)` 一行解决。

#### 3.5.4 流式（Fluid） vs 自适应（Adaptive） vs 响应式（Responsive）

| 方案 | 描述 | 适用场景 |
|---|---|---|
| 流式（Fluid） | 用百分比/`vw`/`fr` 做弹性单位，内容随视口连续缩放 | 图片、文本、Grid 列宽 |
| 自适应（Adaptive） | 在几个固定断点切换布局（768px、1024px 等），无过渡 | 简单网站，设计稿已定好几个尺寸 |
| 响应式（Responsive） | 流式 + 自适应结合：弹性单位打底，关键断点调整结构 | 绝大多数现代项目 |

实际项目中，三者的分界线往往模糊：Grid 的 `fr` 和 `auto-fill` 提供了流式缩放能力，媒体查询负责在特定宽度改变列数或隐藏侧边栏——两者结合就是响应式。

#### 3.5.5 响应式图片

```html
<!-- 根据屏幕宽度加载不同分辨率图片 -->
<img
    srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
    sizes="(max-width: 600px) 100vw, 50vw"
    src="fallback.jpg"
    alt="描述"
>

<!-- 根据设备像素比加载 -->
<img
    srcset="img@1x.jpg 1x, img@2x.jpg 2x"
    src="img@1x.jpg"
    alt="描述"
>
```

在不支持 `srcset` 的旧浏览器中，`src` 作为降级方案。

#### 3.5.6 移动端适配细节

```css
/* 防止 iOS 点击时的半透明遮罩 */
-webkit-tap-highlight-color: transparent;

/* 文字大小调整（防止横屏时文字被放大） */
-webkit-text-size-adjust: 100%;

/* 平滑滚动（iOS 的惯性滚动） */
.scroll-container {
    -webkit-overflow-scrolling: touch;
    overflow-y: auto;
}

/* 安全区域适配（iPhone X 及以上机型） */
.header {
    padding-top: env(safe-area-inset-top);
}
.footer {
    padding-bottom: env(safe-area-inset-bottom);
}
```

## 四、居中方案总结

布局中最常见的需求。根据场景选择合适的方案：

### 水平居中

| 场景 | 方案 |
|---|---|
| 行内/文本元素 | `text-align: center` |
| 定宽块级元素 | `margin: 0 auto` |
| 不定宽块级元素 | 父元素 `display: flex; justify-content: center` |
| 多个元素 | 父元素 Flex + `justify-content: center; gap: 16px` |

### 垂直居中

| 场景 | 方案 |
|---|---|
| 单行文本 | `line-height` 等于容器高度 |
| 定高块级元素 | `position: absolute; top: 50%; margin-top: -50%高度`（需要知道高度） |
| 不定高块级元素 | 父元素 `display: flex; align-items: center` |
| 内容型子元素 | Grid 父容器 + `align-items: center` |

### 水平垂直同时居中

```css
/* 方案一：Flex（推荐——最简洁） */
.parent {
    display: flex;
    justify-content: center;
    align-items: center;
}

/* 方案二：Grid */
.parent {
    display: grid;
    place-items: center;  /* justify-items 和 align-items 的缩写 */
}

/* 方案三：absolute + transform（元素脱离文档流时使用） */
.child {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}
```

## 五、单位一览

| 单位 | 参照 | 使用场景 |
|---|---|---|
| `px` | 设备逻辑像素 | 边框、精确尺寸 |
| `%` | 父元素对应属性 | 流式布局的子元素宽度 |
| `em` | 当前元素 `font-size` | 组件内相对于文字的间距（`padding: 0.5em` 随字号缩放） |
| `rem` | 根元素 `font-size` | 全局一致的间距和字号基准 |
| `vw` / `vh` | 视口宽/高的 1% | 全屏分区、Hero 区域高度 |
| `vmin` / `vmax` | 视口宽高中较小/较大的 1% | 横竖屏切换时保持视觉比例 |
| `fr` | Grid 容器剩余空间 | Grid 列/行宽度分配 |
| `ch` | `0` 字符宽度 | 限制文本栏宽（`max-width: 65ch`） |

`em` vs `rem` 的选择：组件内部需要随字号等比例缩放的间距用 `em`（按钮 padding、标题 margin），跨页面保持全局一致的间距用 `rem`。混用时注意 `em` 的"继承叠加"效应——如果父元素 `font-size: 0.8em`，子元素再写 `font-size: 0.8em`，实际只有根字号的 64%。

## 六、实用组合技巧

### 固定头部 + 可滚动内容 + 固定底部

```css
.page {
    display: flex;
    flex-direction: column;
    height: 100vh; /* 或 100dvh 适配移动端地址栏 */
}
.header, .footer { flex-shrink: 0; height: 56px; }
.content { flex: 1; overflow-y: auto; }
```

`100dvh`（dynamic viewport height）在移动端自动适应地址栏的收起和展开，比 `100vh` 更符合实际。兼容性已覆盖主流现代浏览器。

### 自适应卡片网格（无需媒体查询）

```css
.grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
    gap: 20px;
}
```

核心在于 `auto-fill` + `minmax(280px, 1fr)`：视口足够宽时自动增加列数，太窄时缩为单列。不需要一行媒体查询，但这个方案在卡片尺寸差异大时会有视觉不齐的问题——如果严格需要每行对齐，还是得用媒体查询控制列数。

### 等高列

```css
.columns {
    display: flex;
    /* 默认 align-items: stretch，所有列等高 */
}
.column {
    flex: 1;
}
```

在 Flex 出现之前，等高列需要借助 `display: table-cell` 或巨大的 `padding-bottom` + 负 `margin-bottom` hack。现在只是 Flex 的默认行为。

### 宽高比容器

```css
.video-wrapper {
    aspect-ratio: 16 / 9;
}
.video-wrapper iframe {
    width: 100%;
    height: 100%;
}
```

`aspect-ratio` 是 CSS 里等待最久的属性之一。在此之前只能用 `padding-bottom` 百分比（参照父元素宽度）的 hack。

### 多行文本省略

```css
.clamp-2 {
    display: -webkit-box;
    -webkit-line-clamp: 2;
    -webkit-box-orient: vertical;
    overflow: hidden;
}
```

目前这组属性仍是事实标准（各浏览器都以 `-webkit-` 前缀支持）。单行省略直接用 `text-overflow: ellipsis` + `white-space: nowrap` + `overflow: hidden`。

这份指南覆盖了前端页面布局的主线技术。从最古老的浮动到现代的 Flex/Grid，每个方案都有它适合的场景——没有"过时"的技术，只有用错了场景的方案。理解每种布局的**定位机制**和**约束条件**，比记住一百个 CSS 属性值更重要。
